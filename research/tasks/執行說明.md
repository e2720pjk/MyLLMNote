# 執行流程更新 v2

## 📋 執行方式改進

### 之前的方式（一次呼叫）
- 腳本 → 一次呼叫 OpenCode → 處理所有目標
- 問題：只能看到最終輸出，無法追蹤進度

### 現在的方式（分別呼叫 + 主動通知）
- 腳本 → 分別呼叫每個目標 → 每個完成時通知
- 好處：每項任務完成都會主動通知我

---

## 🔄 新的執行流程

```
腳本執行
  ↓
IP 檢查
  ↓
Goal 001 → OpenCode → 完成 → openclaw message send ↘
Goal 002 → OpenCode → 完成 → openclaw message send  → 我整理 → TG 通知你
Goal 003 → OpenCode → 完成 → openclaw message send  ↗
Goal 004 → OpenCode → 完成 → openclaw message send ↗
  ↓
Goal 005 → OpenCode → 完成 → 每日摘要 + TG 通知用戶
  ↓
腳本結束
```

---

## 📢 通知流程

每個目標完成時：
1. OpenCode 執行：`openclaw message send --target=main "📋 Goal XXX 完成"`
2. 在目標資料夾產出 `results.md`
3. 我收到通知 → 整理 → 回報給你

Goal 005 完成時：
1. 產生每日摘要
2. 執行：`openclaw message send --channel=telegram "🎉 探索任務全部完成！"`
3. 直接透過 Telegram 通知你
