# 子代理回報格式 - 快速參考

## 標準格式模板

```
## 子代理任務完成報告

任務：[簡短描述]
開始時間：[UTC]
結束時間：[UTC]
耗時：[約 X 分鐘]

---

### 📋 執行摘要
[簡述執行過程和主要工作]

---

### 🔍 發現與分析
- 發現 1：[描述]
- 發現 2：[描述]

---

### ✅ 已完成的操作
- [ ] 操作 1
- [ ] 操作 2

---

### ⚠️ 發現的問題 / 阻礙
[無 / 問題描述]

---

### 💬 討論點 / 需要決定
[無 / 決策點]

---

### 🎯 建議的下一步
1. [建議行動]
2. [建議行動]

---

### 📁 相關文件 / 產出
- [文件路徑]
```

---

## 未來使用方式

在 `sessions_spawn` 任務中加入回報要求：

```javascript
{
  task: "任務描述...\n\n## 回報要求\n\n任務完成後，請按照以下格式回報：\n\n[上面模板]\n\n請包含：\n1. 📋 執行摘要\n2. 🔍 發現與分析\n3. ✅ 已完成的操作\n4. ⚠️ 發現的問題 / 阻礙\n5. 💬 討論點 / 需要決定\n6. 🎯 建議的下一步\n7. 📁 相關文件 / 產出"
}
```

這樣就不需要後續確認狀況了！
